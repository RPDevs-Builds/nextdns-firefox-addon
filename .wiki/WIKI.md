### Automation Engine (`sidebars.js`)

To eliminate the overhead of manually updating menus during automated code compilation, sidebars must utilize Docusaurus category autogeneration.

```javascript
// @ts-check

/** @type {import('@docusaurus/plugin-content-docs').SidebarsConfig} */
const sidebars = {
  wikiSidebar: [
    {
      type: 'autogenerated',
      dirName: '.', // Automatically maps the entire /docs folder hierarchy
    },
  ],
};

export default sidebars;
```

## 3. Automated Data Ingestion & Extraction Rules

To keep the wiki tightly coupled to the extension code, execute the following ingestion tasks before compiling the static site assets.

### Task A: Manifest Processing Strategy

1. **Source Input:** Parse `manifest.json` from the main extension development branch.
2. **Transformation Rule:** Extract security parameters, background routing arrays, and injection match profiles into structural Markdown tables.
3. **Target Destination:** Write to `.wiki/dns-forge.github.io/docs/02-architecture/manifest.md`.

### Task B: Code Document Extraction

1. **Source Input:** Analyze all target JS files inside the background engine (`src/background/*.js`) and the modular user interface handlers (`src/ui/*.js`).
2. **Transformation Rule:** Execute `jsdoc-to-markdown` programmatically to convert inline docstrings into strict markdown sections.
3. **Target Destination:** Populate files matching module definitions inside `.wiki/dns-forge.github.io/docs/03-technical-reference/`.

### Task C: Knowledge Alignment

1. **Source Input:** Collect pre-existing foundational specification Markdown documents sitting inside local development paths (e.g., local knowledge maps or `.gemini/` reference directories).
2. **Transformation Rule:** Inject a standardized YAML frontmatter header block into the top of each file containing an explicit `title` and `description`.
3. **Target Destination:** Sync files directly inside targeted subsections within the Docusaurus `/docs/` tree.

## 4. Execution Commands & Build Lifecycle

Manage your wiki operations from the directory root using the following standardized commands:

| Stage / Phase | Pipeline Command | Functional Outcome |
| --- | --- | --- |
| **Dependency Bootstrap** | `npm install` | Restores framework packages and parsing dependencies locally. |
| **Extraction Phase** | `node .tools/generate-wiki-content.js` | Runs the automated parsing, code extraction, and asset layout scripts. |
| **Local Verification** | `npm run start` | Spins up a local web server with Hot Module Replacement at `http://localhost:3000`. |
| **Asset Compilation** | `npm run build` | Validates codebase hyperlinking and builds performance-optimized static assets inside `/build`. |
